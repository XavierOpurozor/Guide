Learning CSS

Welcome to the world of CSS! CSS, or Cascading Style Sheets, is the language used to style HTML elements. Think of it as the "makeup artist" for your website. While HTML provides the structure and content, CSS adds the visual flair â€“ the colors, fonts, layouts, and all the other stylistic details that make a website look appealing. It controls how the HTML elements are displayed in different media (screen, print, etc.).

Applying CSS to HTML

There are three main ways to apply CSS to your HTML:

  1. Inline Styles: Applying styles directly within HTML elements using the style attribute.
  2. Internal Style Sheets: Embedding CSS within the <head> section of your HTML document using the <style> tags.
  3. External Style Sheets: Creating separate .css files and linking them to your HTML document using the <link> tag.

Inline Styles Example

This is the most direct way to style individual HTML elements. You use the style attribute directly within an HTML tag and then specify CSS properties and their values. While quick for small, one-off changes, it can become very messy and hard to maintain for larger projects. It also violates the principle of separation of concerns, making your HTML harder to read and debug. Inline styles also override any styles defined in <style> tags or external stylesheets.

This is a blue heading (Inline)

This is a paragraph with Arial font, a font size of 16 pixels, and a line height of 1.5 (Inline).

When to use inline styles: Rarely. Only for very specific, isolated styling that doesn't need to be reused. For example, if you have a single element on a page that needs a unique style that will never be used elsewhere.

Internal Style Sheets Example

With this method, you embed CSS rules within <style> tags inside the <head> section of your HTML document. These styles will then apply to all the matching elements within that HTML file. This is better than inline styles for applying consistent styles within a single page, but it's still not ideal for styling multiple pages. Internal style sheets are useful when you have a single HTML file and want to keep the styles contained within that file.

In this example, we've added <style> tags within the <head>. Inside the <style> tags, we've defined CSS rules.

This is a green underlined heading (Internal)

This is a special paragraph in italics and purple (Internal).

When to use internal style sheets: When you have a single HTML page and the styles are specific to that page. For example, a single-page application or a quick HTML prototype.

External Style Sheets Example

This involves creating a separate file with a .css extension (e.g., styles.css) and linking it to your HTML document using the <link> element. This is the most organized and reusable way to manage your styles. It promotes a clean separation of concerns, making your HTML and CSS code easier to read, write, and maintain. It also allows for caching of the CSS file by the browser, improving page load times. External style sheets are the standard approach for styling websites.

The <link> tag goes in the <head> of your HTML: <link rel="stylesheet" href="styles.css">

Any CSS rules defined in styles.css will then apply to your HTML. For example, you could create a "styles.css" file with the following content: 


/* styles.css */
body {
  font-family: sans-serif;
  margin: 20px;
}

h1 {
  color: #333;
}

This is the most common way to style web pages. It allows you to keep your HTML structure separate from your CSS styling, making your code easier to manage and update. You can change the look of your entire website by editing a single CSS file!

When to use external style sheets: For virtually all websites and web applications. This is the recommended approach for any project with more than a few HTML pages, or when you want to reuse styles across multiple pages.

CSS Selectors

CSS selectors are used to target the specific HTML elements you want to style. We've already seen some basic selectors. Here's a more detailed look:

  • Element Selectors (Type Selectors): Select elements based on their tag name (e.g., p, h1, body). For example, p { ... } styles all paragraph elements.
  • Class Selectors: Select elements with a specific class attribute (e.g., .special-paragraph). You can apply the same style to multiple elements. For example, .my-class { ... } styles all elements with the class "my-class".
  • ID Selectors: Select a specific element with a unique ID attribute (e.g., #my-unique-element). IDs should be unique within a page. For example, #header { ... } styles the element with the ID "header".
  • Universal Selector: Selects all elements (e.g., * { ... }). Often used for resetting default styles.
  • Attribute Selectors: Select elements based on their attributes or attribute values (e.g., a[href], input[type="text"]).
  • Pseudo-classes: Select elements based on their state or position (e.g., a:hover, :first-child).
  • Pseudo-elements: Create virtual elements to style specific parts of an element (e.g., p::first-letter, div::before).
  • Combinators: Define relationships between selectors (e.g., descendant selector, child selector, adjacent sibling selector, general sibling selector).

Example of Selectors

Here's how you might use these selectors in your CSS:


/* styles.css */
body {
  font-family: sans-serif;  /* Element Selector */
}

h1 {
  color: #333;           /* Element Selector */
}

.special-paragraph {
  font-style: italic;    /* Class Selector */
}

#my-unique-element {
  font-size: 24px;      /* ID Selector */
}

* {
  box-sizing: border-box; /* Universal Selector */
}

a[href] {
  color: #0078d7;       /* Attribute Selector */
  text-decoration: none;
}

a:hover {
  text-decoration: underline;  /* Pseudo-class */
}

p::first-letter {
  font-size: 150%;        /* Pseudo-element */
  font-weight: bold;
}

div p {
  color: green;          /* Descendant Selector */
}

ul > li {
  list-style-type: square; /* Child Selector */
}

More on Selectors

Let's dig deeper into some of the more advanced selectors:

Attribute Selectors

Attribute selectors allow you to target elements based on the presence or value of their attributes.

  • [attribute]: Selects elements that have the specified attribute. For example, a[href] selects all <a> elements with a href attribute.
  • [attribute="value"]: Selects elements with the specified attribute and value. For example, input[type="text"] selects all <input> elements with type="text".
  • [attribute~="value"]: Selects elements whose attribute value contains the specified value as part of a whitespace-separated list. For example, div[class~="warning"] selects all <div> elements whose class attribute contains "warning" (e.g., class="warning message").
  • [attribute|="value"]: Selects elements whose attribute value is exactly "value" or begins with "value-" (e.g., lang="en" or lang="en-US").
  • [attribute^="value"]: Selects elements whose attribute value begins with the specified value.
  • [attribute$="value"]: Selects elements whose attribute value ends with the specified value.
  • [attribute*="value"]: Selects elements whose attribute value contains the specified value anywhere within the string.

Example:

Pseudo-classes

Pseudo-classes select elements based on their state or position, not based on attributes or content. They begin with a colon :.

  • :hover: Selects an element when the user hovers over it.
  • :active: Selects an element when it is being activated (e.g., when a button is clicked).
  • :focus: Selects an element that has focus (e.g., an input field that is selected).
  • :first-child: Selects the first child element of another element.
  • :last-child: Selects the last child element of another element.
  • :nth-child(n): Selects the nth child element of another element (where n can be a number, keyword, or formula).
  • :nth-of-type(n): Selects the nth element of a specific type that is a child of another element.
  • :not(selector): Selects elements that do not match the specified selector.
  • :checked: Selects checked checkboxes or radio buttons.
  • :disabled: Selects disabled form elements.
  • :enabled: Selects enabled form elements.
  • :valid: selects form elements with valid values
  • :invalid: selects form elements with invalid values

  font-weight: bold;

Pseudo-elements

Pseudo-elements create virtual elements to style specific parts of an element. They begin with a double colon ::.

  • ::first-letter: Selects the first letter of the text in an element.
  • ::first-line: Selects the first line of the text in an element.
  • ::before: Inserts content before the content of an element.
  • ::after: Inserts content after the content of an element.
  • ::selection: Applies styles to the selected portion of an element.

Combinators

Combinators define the relationship between selectors.

  • Descendant Selector (space): Selects elements that are descendants of another element (e.g., div p selects all <p> elements that are inside a <div> element).
  • Child Selector (>): Selects elements that are direct children of another element (e.g., ul > li selects all <li> elements that are direct children of a <ul> element).
  • Adjacent Sibling Selector (+): Selects an element that is the next sibling of another element (e.g., h1 + p selects the first <p> element that immediately follows an <h1> element).
  • General Sibling Selector (~): Selects elements that are siblings of another element, but not necessarily immediately following it (e.g., h1 ~ p selects all <p> elements that are siblings of an <h1> element).

Example:

  color: blue;
}
ul > li {
  font-weight: bold;
}
h1 + p {
  text-decoration: underline;
}
h1 ~ p {
  color: green;
}

CSS Box Model

The CSS box model is a fundamental concept that describes how HTML elements are rendered on a page. Every element can be thought of as a rectangular box, and styling these boxes is crucial for creating layouts. The box model consists of:

  • Content: The actual content of the element (text, images, etc.). The size of the content area can be set using the width and height properties.
  • Padding: The space between the content and the border. Padding is controlled by the padding property, and can be set for all four sides at once, or individually using padding-top, padding-right, padding-bottom, and padding-left.
  • Border: A line that surrounds the padding and content. The border is controlled by the border property, which is a shorthand for setting the border-width, border-style, and border-color. You can also set these properties individually, and set border properties for individual sides (e.g., border-top-width).
  • Margin: The space outside the border, separating the element from other elements. Margin is controlled by the margin property, and can be set for all four sides at once, or individually using margin-top, margin-right, margin-bottom, and margin-left. Margins can collapse vertically.

Example: 


.box-model-example {
  width: 300px;
  height: 150px;
  padding: 20px;
  border: 5px solid #0078d7;
  margin: 30px;
  background-color: #f0f4f8;
}

CSS Layout

CSS provides several properties and techniques for controlling the layout of elements on a webpage. Here are some key concepts:

Display Property

The display property specifies the display behavior of an element. Common values include:

  • block: The element takes up the full width available and starts a new line (e.g., <div>, <p>).
  • inline: The element only takes up as much width as necessary and does not start a new line (e.g., <span>, <a>).
  • inline-block: A combination of inline and block; the element flows like inline but can have block-level properties like width and height.
  • none: The element is not displayed.
  • flex: Enables the Flexbox layout model.
  • grid: Enables the Grid layout model.

Position Property

The position property controls how an element is positioned within its containing element.

  • static: The default value. Elements are positioned according to the normal flow of the document. The top, right, bottom, and left properties have no effect on statically positioned elements.
  • relative: The element is positioned relative to its normal position. The top, right, bottom, and left properties can be used to offset the element from its normal position. Other elements are not affected by the offset.
  • absolute: The element is positioned relative to its nearest positioned ancestor (an ancestor with a position other than static). If there is no positioned ancestor, it is positioned relative to the initial containing block (the <html> element). Absolutely positioned elements are removed from the normal flow of the document, and can overlap other elements.
  • fixed: The element is positioned relative to the viewport (the browser window) and stays in place even when the page is scrolled. Fixed positioning is useful for creating elements that remain visible regardless of scroll position, such as navigation bars or chat windows.
  • sticky: The element is positioned based on the user's scroll position. It's relative until it reaches a specified offset, then it becomes fixed. A sticky element toggles between relative and fixed, depending on the scroll position.

Flexbox

Flexbox is a powerful layout model for creating one-dimensional layouts (rows or columns). It provides properties for controlling the alignment, direction, and spacing of items within a container. Flexbox is particularly useful for aligning elements within a container, even when their sizes are unknown or dynamic.

Key Flexbox properties:

  • display: flex;: Defines the container as a flex container, enabling flexbox properties.
  • flex-direction: Specifies the direction of the flex items within the container (row, row-reverse, column, column-reverse).
  • justify-content: Aligns flex items along the main axis (horizontal for rows, vertical for columns) (flex-start, flex-end, center, space-between, space-around, space-evenly).
  • align-items: Aligns flex items along the cross axis (vertical for rows, horizontal for columns) (flex-start, flex-end, center, baseline, stretch).
  • flex-wrap: Specifies whether the flex items should wrap if they don't fit in the container (nowrap, wrap, wrap-reverse).
  • flex: A shorthand property for setting flex-grow, flex-shrink, and flex-basis.
  • flex-grow: Specifies how much the item will grow relative to the other flex items in the container.
  • flex-shrink: Specifies how much the item will shrink relative to the other flex items in the container.
  • flex-basis: Specifies the initial size of the flex item before any growing or shrinking occurs.
  • align-self: Overrides the align-items property for a specific flex item.
  • order: Specifies the order of a flex item within the flex container.

Example:


.flex-container {
  display: flex;
  flex-direction: row;

Grid

Grid is a two-dimensional layout model that allows you to create complex, responsive layouts with rows and columns. It offers more control than Flexbox for multi-dimensional layouts. Grid is excellent for creating page layouts, form layouts, and any layout where you need precise control over the positioning of elements in both rows and columns.

Key Grid properties:

  • display: grid;: Defines the element as a grid container.
  • grid-template-rows: Specifies the height of the rows in the grid container.
  • grid-template-columns: Specifies the width of the columns in the grid container.
  • grid-template-areas: Defines the layout of the grid by referencing the names of the grid items.
  • grid-row-gap: Specifies the gap between rows. Renamed to row-gap in newer CSS specifications.
  • grid-column-gap: Specifies the gap between columns. Renamed to column-gap in newer CSS specifications.
  • gap: A shorthand property for setting both row-gap and column-gap.
  • justify-items: Aligns grid items along the inline axis (horizontally in a row-direction grid, vertically in a column-direction grid).
  • align-items: Aligns grid items along the block axis (vertically in a row-direction grid, horizontally in a column-direction grid).
  • justify-content: Aligns the grid within the grid container along the inline axis.
  • align-content: Aligns the grid within the grid container along the block axis.
  • grid-auto-rows: Specifies the size of implicitly created rows.
  • grid-auto-columns: Specifies the size of implicitly created columns.
  • grid-auto-flow: Controls how auto-placed items are inserted into the grid.
  • grid-row-start, grid-row-end, grid-column-start, grid-column-end: Specify the starting and ending lines of a grid item within the grid.
  • grid-area: A shorthand property for setting grid-row-start, grid-column-start, grid-row-end, and grid-column-end.

CSS Styling Text

CSS provides many properties for styling the text within HTML elements:

  • font-family: Specifies the font to use (e.g., "Arial", "Times New Roman", "sans-serif"). You can specify a list of font families as a fallback in case the primary font is not available.
  • font-size: Sets the size of the font (e.g., "16px", "1.2em", "100%"). Use relative units like em or rem for better accessibility and responsiveness.
  • font-weight: Sets the weight (boldness) of the font (e.g., "normal", "bold", "bolder", "lighter", or numeric values like "400", "700").
  • font-style: Sets the style of the font (e.g., "normal", "italic", "oblique").
  • color: Sets the color of the text. Colors can be specified using keywords, hex codes, RGB, RGBA, HSL, or HSLA values.
  • text-align: Aligns the text within its container (e.g., "left", "center", "right", "justify").
  • text-decoration: Adds or removes decoration from text (e.g., "none", "underline", "overline", "line-through").
  • line-height: Sets the vertical space between lines of text. A value without a unit (e.g., 1.5) multiplies the font size.
  • letter-spacing: Sets the horizontal spacing between characters.
  • word-spacing: Sets the horizontal spacing between words.
  • text-shadow: Adds a shadow to the text. You can specify the horizontal offset, vertical offset, blur radius, and color of the shadow.
  • text-transform: Controls the capitalization of text (e.g., "none", "capitalize", "uppercase", "lowercase").
  • white-space: Controls how whitespace within an element is handled (e.g., "normal", "nowrap", "pre", "pre-wrap", "pre-line").

Example:


.text-example {
  font-family: "Arial", sans-serif;
  font-size: 1.2em;
  font-weight: bold;
  font-style: italic;
  color: #0078d7;
  text-align: center;
  text-decoration: underline;
  line-height: 1.5;
  letter-spacing: 2px;
  word-spacing: 5px;
  text-shadow: 2px 2px 4px rgba(0, 0, 0, 0.2);
  text-transform: uppercase;
  white-space: pre-wrap;
}

CSS Transitions and Animations

CSS allows you to create dynamic and engaging web pages with transitions and animations:

Transitions

Transitions allow you to smoothly change CSS properties over a specified duration when an element changes its state (e.g., when hovered). You define the properties you want to transition, the duration of the transition, and the timing function (which controls the speed curve of the transition).

Key transition properties:

  • transition-property: Specifies the CSS property or properties to which the transition effect should be applied (e.g., all, background-color, transform).
  • transition-duration: Specifies the length of time a transition should take to complete (e.g., 0.3s, 200ms).
  • transition-timing-function: Specifies the speed curve of the transition effect (e.g., ease, linear, ease-in, ease-out, ease-in-out, cubic-bezier()).
  • transition-delay: Specifies a delay before the transition effect starts.
  • transition: A shorthand property for setting all of the above properties.

Example:


.transition-example {
  width: 100px;
  height: 100px;
  background-color: #0078d7;
  transition: background-color 0.5s ease-in-out, transform 0.5s ease-in-out;
}
.transition-example:hover {
  background-color: #fef08a;
  transform: rotate(180deg);
}
    

<div class="transition-example">Hover Me</div>

Animations

Animations allow you to create more complex sequences of CSS property changes over time. You define keyframes to control the animation at different points. Animations provide more control than transitions, allowing you to create looping animations and control the animation at specific points in its sequence.

Key animation properties:

  • @keyframes: Defines the sequence of keyframes for an animation. Keyframes specify the styles at different points during the animation.
  • animation-name: Specifies the name of the @keyframes rule that defines the animation's sequence.
  • animation-duration: Specifies the length of time an animation should take to complete.
  • animation-timing-function: Specifies the speed curve of the animation effect.
  • animation-delay: Specifies a delay before the animation starts.
  • animation-iteration-count: Specifies the number of times an animation should play (e.g., 1, infinite).
  • animation-direction: Specifies whether an animation should play forwards, backwards, or in alternate cycles.
  • animation-fill-mode: Specifies how an element should be styled when an animation is not playing (before it starts, after it ends).
  • animation: A shorthand property for setting all of the above animation properties.

Example:


.animation-example {
  width: 100px;
  height: 100px;
  background-color: #0078d7;
  animation-name: changeColorAndSize;
  animation-duration: 2s;
  animation-iteration-count: infinite;
  animation-timing-function: ease-in-out;
}
@keyframes changeColorAndSize {
  0% {
    background-color: #0078d7;
    transform: scale(1);
  }
  50% {
    background-color: #fef08a;
    transform: scale(1.5);
  }
  100% {
    background-color: #0078d7;
    transform: scale(1);
  }
}
    

<div class="animation-example">Animate Me</div>

Responsive Design

Responsive web design is the practice of designing websites to adapt to different screen sizes and devices (desktops, tablets, smartphones). CSS plays a crucial role in responsive design.

Media Queries

Media queries allow you to apply different CSS rules based on the characteristics of the user's device, such as its width, height, and orientation. They allow you to create different layouts and styles for different devices, ensuring a consistent and user-friendly experience across all platforms.

Media queries consist of a media type (e.g., all, screen, print) and one or more media features (e.g., max-width, min-width, orientation).

Example:


body {
  font-size: 16px;
}
@media (max-width: 768px) {
  body {
    font-size: 14px;
  }
  .container {
    width: 100%;
    padding: 10px;
  }
}
@media (min-width: 1200px) {
  body {
    font-size: 18px;
  }
  .container {
    width: 1170px;
    margin: 0 auto;
  }
}
    

<div class="container">
  <p>
    This website uses media queries to adjust the font size and container width based on the screen size.  On small screens (less than 768px), the font size is 14px and the container takes up the full width.  On large screens (1200px and wider), the font size is 18px and the container has a fixed width of 1170px and is centered.
  </p>
</div>

Flexible Units

Using relative units like percentages (%), em, and rem instead of fixed units like pixels (px) helps elements scale more effectively on different screens. Relative units allow your design to adapt to the size of the viewport or the user's font settings, making your website more accessible and responsive.

  • %: Specifies a size as a percentage of the element's containing element.
  • em: Specifies a size relative to the font size of the element itself.
  • rem: Specifies a size relative to the font size of the root element (<html>).
  • vw: Specifies a size relative to 1% of the viewport's width.
  • vh: Specifies a size relative to 1% of the viewport's height.

Example:


.container {
  width: 90%;
  max-width: 1200px;
  margin: 0 auto;
  padding: 1em;
}
h2 {
  font-size: 2rem;
  margin-bottom: 1em;
}
p {
  font-size: 1em;
  line-height: 1.5;
}
    

<div class="container">
  <h2>Responsive Design Example</h2>
  <p>
    This container has a width of 90% of its parent, but a maximum width of 1200px.  The heading font size is 2rem (2 times the root font size), and the paragraph font size is 1em (the same as the element's font size).
  </p>
</div>

Viewport Meta Tag

The <meta name="viewport" content="width=device-width, initial-scale=1.0"> tag in the <head> is essential for ensuring that your page scales correctly on mobile devices. It tells the browser how to control the page's scaling and dimensions.

Key attributes of the viewport meta tag:

  • width=device-width: Sets the width of the viewport to the width of the device.
  • initial-scale=1.0: Sets the initial zoom level when the page is first loaded.
  • maximum-scale: Sets the maximum zoom level.
  • minimum-scale: Sets the minimum zoom level.
  • user-scalable: Specifies whether the user can zoom in or out (yes or no).

Example:


<!DOCTYPE html>
<html>
<head>
  <title>Responsive Page</title>
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <style>
    body {
      font-size: 16px;
      padding: 10px;
    }
    @media (max-width: 480px) {
      body {
        font-size: 12px;
        padding: 5px;
      }
    }
  </style>
</head>
<body>
  <h1>Responsive Design</h1>
  <p>
    This page uses the viewport meta tag to ensure it is displayed correctly on mobile devices. The font size and padding adjust based on the screen width.
  </p>
</body>
</html>
