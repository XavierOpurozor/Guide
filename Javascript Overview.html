Comprehensive JavaScript Tutorial

This tutorial provides a comprehensive introduction to JavaScript, covering fundamental concepts to more advanced topics. You can run the JavaScript code snippets directly in your browser's developer console (usually opened by pressing F12).

1. Introduction to JavaScript

JavaScript is a high-level, interpreted programming language that is primarily used to make web pages interactive. It is one of the three core technologies of the World Wide Web, alongside HTML and CSS.

2. Setting Up

You don't need any special software to start with JavaScript. You can write and run JavaScript code directly in your web browser's developer console or by embedding it within HTML <script> tags.

<!DOCTYPE html>
<html>
<head>
    <title>My First JavaScript</title>
</head>
<body>
    <h1>Hello World!</h1>
    <p id="demo"></p>

    <script>
        document.getElementById("demo").innerHTML = "JavaScript is running!";
    </script>
</body>
</html>

3. Basic Syntax

JavaScript syntax is similar to many other programming languages. Here are some fundamental elements:

  • Statements: Instructions that perform an action, usually ending with a semicolon (;).
  • Variables: Used to store data values. Declared using var, let, or const.
  • Operators: Symbols used to perform operations on values (e.g., +, -, *, /, =).
  • Comments: Used to explain code. Single-line comments start with //, and multi-line comments are enclosed in /* ... */.
// This is a single-line comment
/*
This is a
multi-line comment
*/

let message = "Hello"; // Declare a variable using 'let'
const PI = 3.14159;   // Declare a constant using 'const'
var oldVariable = "This is older syntax"; // Declare a variable using 'var' (less common now)

let sum = 5 + 3; // Addition operator
let product = 5 * 3; // Multiplication operator

4. Variables

Variables are containers for storing data. JavaScript has three keywords for declaring variables:

  • var: Older way to declare variables. Has function scope.
  • let: Introduced in ES6 (ECMAScript 2015). Has block scope. Recommended for most cases.
  • const: Also introduced in ES6. Used for variables whose values should not be reassigned. Also has block scope.
let name = "John Doe";
let age = 30;
const isStudent = false;

name = "Jane Doe"; // You can reassign 'let' variables
// isStudent = true; // This would cause an error because 'const' variables cannot be reassigned

if (true) {
    let blockScopedVar = "I'm inside the block";
    var functionScopedVar = "I'm function scoped";
    console.log(blockScopedVar); // Accessible here
}
// console.log(blockScopedVar); // Not accessible here (block scope)
console.log(functionScopedVar); // Accessible here (function scope)

5. Data Types

JavaScript has several built-in data types:

  • Primitive Data Types:
    • String: Represents textual data (e.g., "Hello").
    • Number: Represents numeric data (integers and floating-point numbers, e.g., 10, 3.14).
    • Boolean: Represents logical values (true or false).
    • Null: Represents the intentional absence of a value.
    • Undefined: Represents a variable that has been declared but has not been assigned a value.
    • Symbol (ES6): Represents a unique identifier.
    • BigInt (ES2020): Represents integers of arbitrary length.
  • Object Data Type:
    • Object: A collection of key-value pairs.
    • Array: An ordered list of values.
    • Function: A block of code designed to perform a particular task.
    • Date: Represents a specific point in time.
    • And more...

6. Operators

JavaScript operators are symbols that perform operations on operands (values and variables).

  • Arithmetic Operators: + (addition), - (subtraction), * (multiplication), / (division), % (modulus), ** (exponentiation), ++ (increment), -- (decrement).
  • Assignment Operators: = (assign), +=, -=, *=, /=, %=, **=.
  • Comparison Operators: == (equal to), === (strict equal to), != (not equal to), !== (strict not equal to), > (greater than), < (less than), >= (greater than or equal to), <= (less than or equal to).
  • Logical Operators: && (logical AND), || (logical OR), ! (logical NOT).
let a = 10;
let b = 5;

console.log(a + b);   // Output: 15
console.log(a - b);   // Output: 5
console.log(a * b);   // Output: 50
console.log(a / b);   // Output: 2
console.log(a % b);   // Output: 0
console.log(a ** 2);  // Output: 100

let c = 7;
c += 3; // c is now 10

console.log(a == b);  // Output: false (checks only value)
console.log(a === "10"); // Output: false (checks value and type)
console.log(a != b);  // Output: true
console.log(a !== "10");// Output: true

let x = true;
let y = false;

console.log(x && y);  // Output: false
console.log(x || y);  // Output: true
console.log(!x);      // Output: false

7. Control Flow

Control flow statements determine the order in which code is executed.

  • if, else if, else: Conditional statements.
  • switch: Selects one of many code blocks to be executed.
  • for: Loops through a block of code a number of times.
  • while: Loops through a block of code while a condition is true.
  • do...while: Similar to while, but the code block is executed at least once.
  • break: Exits a loop or a switch statement.
  • continue: Skips the current iteration of a loop and continues to the next.
let ageToCheck = 20;

if (ageToCheck >= 18) {
    console.log("You are an adult.");
} else if (ageToCheck >= 13) {
    console.log("You are a teenager.");
} else {
    console.log("You are a child.");
}

let day = "Monday";
switch (day) {
    case "Monday":
        console.log("It's Monday!");
        break;
    case "Friday":
        console.log("It's almost the weekend!");
        break;
    default:
        console.log("It's another day.");
}

for (let i = 0; i < 5; i++) {
    console.log("Loop iteration: " + i);
}

let count = 0;
while (count < 3) {
    console.log("While loop: " + count);
    count++;
}

let j = 0;
do {
    console.log("Do...while loop: " + j);
    j++;
} while (j < 2);

13. Asynchronous JavaScript and Callbacks

JavaScript is single-threaded, meaning it executes code one line at a time. Asynchronous operations allow you to perform long-running tasks without blocking the main thread, ensuring the user interface remains responsive. Callbacks are a way to handle the results of asynchronous operations.

<!DOCTYPE html>
<html>
<head>
    <title>Async JavaScript</title>
</head>
<body>
    <div id="output"></div>

    <script>
        // Example 1: setTimeout()
        console.log("Before setTimeout");
        setTimeout(function() {
            console.log("Inside setTimeout (after 2 seconds)");
            document.getElementById("output").textContent = "Delayed message!";
        }, 2000); // 2000 milliseconds = 2 seconds
        console.log("After setTimeout");
        //output:
        // Before setTimeout
        // After setTimeout
        // Inside setTimeout (after 2 seconds)


        // Example 2:  Callback function with setTimeout
        function fetchData(callback) {
            setTimeout(function() {
                const data = { name: "John", age: 30 };
                callback(data); // Call the callback function with the data
            }, 1500);
        }

        fetchData(function(result) {
            console.log("Data received:", result);
            document.getElementById("output").textContent += "<br>Data received: " + result.name + ", " + result.age;
        });

    </script>
</body>
</html>

14. Promises

Promises are a more modern way to handle asynchronous operations in JavaScript. They provide a cleaner syntax and better error handling compared to callbacks. A Promise represents the eventual completion (or failure) of an asynchronous operation and its resulting value.

<!DOCTYPE html>
<html>
<head>
    <title>Promises in JavaScript</title>
</head>
<body>
    <div id="output"></div>

    <script>
        // Creating a Promise
        const myPromise = new Promise(function(resolve, reject) {
            // Simulate an asynchronous operation (e.g., fetching data)
            setTimeout(function() {
                const success = true; // Or false
                if (success) {
                    resolve({ message: "Operation succeeded!", data: [1, 2, 3] }); // Resolve with a value
                } else {
                    reject("Operation failed!"); // Reject with a reason
                }
            }, 2000);
        });

        // Consuming the Promise
        myPromise
            .then(function(result) {
                console.log("Result:", result);
                document.getElementById("output").textContent = result.message + "<br>Data: " + result.data.join(", ");
            })
            .catch(function(error) {
                console.error("Error:", error);
                document.getElementById("output").textContent += "<br>Error: " + error;
            });

        // Promise.resolve and Promise.reject
        const resolvedPromise = Promise.resolve("Already resolved!");
        resolvedPromise.then(value => console.log(value));  //Output: Already resolved!

        const rejectedPromise = Promise.reject("Rejected immediately!");
        rejectedPromise.catch(error => console.error(error)); // Output: Rejected immediately!

    </script>
</body>
</html>

15. Async/Await

async and await are syntactic sugar built on top of Promises, making asynchronous code even easier to write and read. The async keyword is used to define an asynchronous function, and the await keyword is used to pause the execution of that function until a Promise is resolved.

<!DOCTYPE html>
<html>
<head>
    <title>Async/Await in JavaScript</title>
</head>
<body>
    <div id="output"></div>

    <script>
        // Function that returns a Promise (simulates fetching data)
        function fetchData() {
            return new Promise(resolve => {
                setTimeout(() => {
                    resolve({ name: "Alice", age: 28 });
                }, 1500);
            });
        }

        // Using async/await
        async function getData() {
            try {
                console.log("Start fetching data...");
                document.getElementById("output").textContent = "Start fetching data...";
                const result = await fetchData(); // Wait for the Promise to resolve
                console.log("Data received:", result);
                document.getElementById("output").textContent += "<br>Data received: " + result.name + ", " + result.age;
            } catch (error) {
                console.error("Error:", error);
                document.getElementById("output").textContent += "<br>Error: " + error;
            } finally {
                console.log("Fetch operation completed.");
document.getElementById("output").textContent += "<br>Fetch operation completed.";
            }
        }

        // Call the async function
        getData();
    </script>
</body>
</html>

16. Error Handling

Error handling is crucial for writing robust JavaScript code. You can use try...catch...finally blocks to handle exceptions (errors) that may occur during the execution of your code.

<!DOCTYPE html>
<html>
<head>
    <title>Error Handling in JavaScript</title>
</head>
<body>
    <div id="output"></div>

    <script>
        function divide(a, b) {
            if (b === 0) {
                throw new Error("Division by zero is not allowed."); // Throw an error
            }
            return a / b;
        }

        try {
            let result1 = divide(10, 2);
            console.log("Result 1:", result1); // Output: 5
            document.getElementById("output").textContent = "Result 1: " + result1;

            let result2 = divide(5, 0); // This will throw an error
            console.log("Result 2:", result2); // This line will not be reached
            document.getElementById("output").textContent += "<br>Result 2: " + result2;

        } catch (error) {
            console.error("An error occurred:", error.message); // Catch the error and handle it
            document.getElementById("output").textContent += "<br>Error: " + error.message;
        } finally {
            console.log("Finally block executed."); // This block always executes, regardless of whether an error occurred
            document.getElementById("output").textContent += "<br>Finally block executed.";
        }
    </script>
</body>
</html>

17. Modules

JavaScript modules allow you to split your code into separate files, making it more organized and maintainable. You can use import and export statements to share code between modules.

To run this example, you'll need a server (like Node.js) or a bundler (like Webpack) to handle the module syntax. Browsers don't fully support modules without these tools in simple file setups.


        // math.js (Create this file separately)
        export const add = (a, b) => a + b;
        export const subtract = (a, b) => a - b;

        // main.js (Create this file separately)
        import { add, subtract } from './math.js';  // Corrected import path
        console.log(add(5, 3));       // Output: 8
        console.log(subtract(10, 4));  // Output: 6

Explanation and how to run with a simple server (Node.js):

  1. Create Files: Create two files: math.js and main.js, and copy the code above into them. Make sure they are in the same directory.
  2. Install Node.js: If you don't have it, download and install Node.js from nodejs.org.
  3. Create package.json: In the directory containing your files, open a terminal and run npm init -y. This creates a package.json file.
  4. Install a simple server (http-server): In the same terminal, run npm install http-server -g
  5. Run the server: In the same terminal, run npx http-server -p 8000. This will start a simple web server.
  6. Create an HTML file: Create an index.html file in the same directory: 
    
<!DOCTYPE html>
<html>
<head>
    <title>Modules Example</title>
    <script src="main.js" type="module"></script> <!-- Important: type="module" -->
</head>
<body>
    <div id="output"></div>
</body>
</html>
  7. Open in Browser: Open your browser and go to http://localhost:8000. You should see the output from main.js in the console.

18. Classes

Classes in JavaScript provide a more structured way to create objects. They are syntactic sugar over JavaScript's prototype-based inheritance.


<!DOCTYPE html>
<html>
<head>
  <title>Classes in JavaScript</title>
</head>
<body>
  <div id="output"></div>
  <script>
    // Basic class definition
    class Person {
      constructor(name, age) {
        this.name = name;
        this.age = age;
      }

      greet() {
        console.log(`Hello, my name is ${this.name} and I am ${this.age} years old.`);
        document.getElementById("output").innerHTML += `Hello, my name is ${this.name} and I am ${this.age} years old.<br>`;
      }
    }

    // Creating instances
    let person1 = new Person("John", 30);
    person1.greet();

    let person2 = new Person("Jane", 25);
    person2.greet();

    // Inheritance
    class Student extends Person {
      constructor(name, age, major) {
        super(name, age);
        this.major = major;
      }

      study() {
        console.log(`${this.name} is studying ${this.major}.`);
        document.getElementById("output").innerHTML += `${this.name} is studying ${this.major}.<br>`;
      }

      greet() {
        console.log(`Hello, I am ${this.name}, a student of ${this.major}.`);
        document.getElementById("output").innerHTML += `Hello, I am ${this.name}, a student of ${this.major}.<br>`;
      }
    }

    let student1 = new Student("Alice", 20, "Computer Science");
    student1.greet();
    student1.study();
  </script>
</body>
</html>

20. Fetch API

The Fetch API provides a modern, flexible interface for making HTTP requests. It can be used to retrieve data from a server, send data to a server, and more. It uses Promises to handle asynchronous operations.


<!DOCTYPE html>
<html>
<head>
  <title>Fetch API in JavaScript</title>
</head>
<body>
  <div id="output" style="border: 1px solid #ccc; padding: 10px; margin-bottom: 10px;"></div>
  <div id="postOutput" style="border: 1px solid #ccc; padding: 10px;"></div>

  <script>
    const outputDiv = document.getElementById('output');
    const postOutputDiv = document.getElementById('postOutput');

    // Example 1: GET request
    fetch('https://jsonplaceholder.typicode.com/todos/1')
      .then(response => {
        if (!response.ok) throw new Error(`HTTP error! status: ${response.status}`);
        return response.json();
      })
      .then(data => {
        console.log('GET Response:', data);
        outputDiv.innerHTML += `<p><b>GET Response:</b> Title: ${data.title}, Completed: ${data.completed}</p>`;
      })
      .catch(error => {
        console.error('GET Error:', error);
        outputDiv.innerHTML += `<p style="color:red;"><b>GET Error:</b> ${error}</p>`;
      });

    // Example 2: POST request
    fetch('https://jsonplaceholder.typicode.com/posts', {
      method: 'POST',
      headers: {
        'Content-type': 'application/json; charset=UTF-8',
      },
      body: JSON.stringify({
        title: 'foo',
        body: 'bar',
        userId: 1,
      }),
    })
      .then(response => {
        if (!response.ok) throw new Error(`HTTP error! status: ${response.status}`);
        return response.json();
      })
      .then(data => {
        console.log('POST Response:', data);
        postOutputDiv.innerHTML += `<p><b>POST Response:</b> ID: ${data.id}, Title: ${data.title}</p>`;
      })
      .catch(error => {
        console.error('POST Error:', error);
        postOutputDiv.innerHTML += `<p style="color:red;"><b>POST Error:</b> ${error}</p>`;
      });
  </script>
</body>
</html>

Note: The API endpoints used in the examples are from jsonplaceholder.typicode.com, a free online REST API for testing and prototyping.

21. WebSockets

WebSockets provide a way to establish a persistent, bidirectional, and low-latency communication channel between a client and a server. This enables real-time applications, such as chat, online games, and live data dashboards.

This example requires a WebSocket server. You can use a Node.js library like ws to create one. The client-side code can run in any browser that supports WebSockets. 


        <!DOCTYPE html>
        <html>
        
            WebSocket Example
        
        
            <div id="output" style="border: 1px solid #ccc; padding: 10px; margin-bottom: 10px; height: 200px; overflow-y: auto;"></div>
            <input type="text" id="messageInput" placeholder="Enter message">
            <button id="sendButton">Send</button>

            <script>
                const outputDiv = document.getElementById('output');
                const messageInput = document.getElementById('messageInput');
                const sendButton = document.getElementById('sendButton');

                // 1. Create a WebSocket connection
                //    Replace 'ws://localhost:8080' with your WebSocket server URL
                const socket = new WebSocket('ws://localhost:8080');

                // 2.  Handle the connection open event
                socket.addEventListener('open', (event) => {
                    console.log('WebSocket connection opened:', event);
                    outputDiv.innerHTML += '<p><b>[Connected]</b> Connection established!</p>';
                    // Send a message to the server after the connection is opened
                    socket.send('Hello Server!');
                });

                // 3. Handle incoming messages from the server
                socket.addEventListener('message', (event) => {
                    console.log('Received message:', event.data);
                    outputDiv.innerHTML += `<p><b>Server:</b> ${event.data}</p>`;
                });

                // 4. Handle the connection close event
                socket.addEventListener('close', (event) => {
                    console.log('WebSocket connection closed:', event);
                    outputDiv.innerHTML += '<p><b>[Disconnected]</b> Connection closed.</p>';
                });

                // 5. Handle errors
                socket.addEventListener('error', (event) => {
                    console.error('WebSocket error:', event);
                    outputDiv.innerHTML += `<p style="color:red;"><b>[Error]</b> An error occurred: ${event.message}</p>`;
                });

                // 6. Send a message when the send button is clicked
                sendButton.addEventListener('click', () => {
                    const message = messageInput.value;
                    if (message.trim() !== '') {
                        console.log('Sending message:', message);
                        outputDiv.innerHTML += `<p><b>Client:</b> ${message}</p>`;
                        socket.send(message); // Send the message to the server
                        messageInput.value = ''; // Clear the input field
                    }
                });

To make this example work, you would need a WebSocket server running at ws://localhost:8080. Here's a simple Node.js WebSocket server using the ws library: 


        // server.js (Node.js)
        const WebSocket = require('ws');

        const wss = new WebSocket.Server({ port: 8080 });

        wss.on('connection', (ws) => {
            console.log('Server: Client connected');

            // Send a message to the client upon connection
            ws.send('Welcome to the WebSocket server!');

            // Handle messages from the client
            ws.on('message', (message) => {
                console.log('Server received:', message);

                // Echo the message back to the client
                ws.send(`Server received: ${message}`);
            });

            // Handle the client disconnecting
            ws.on('close', () => {
                console.log('Server: Client disconnected');
            });

            //Handle errors
            ws.on('error', (error)=>{
                console.error('Server: error', error);
            })
        });

        console.log('WebSocket server is running on port 8080');
